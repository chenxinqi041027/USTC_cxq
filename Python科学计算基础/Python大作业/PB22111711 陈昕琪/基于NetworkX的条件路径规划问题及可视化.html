1. 科学技术原理

问 题引入：蚂蚁的最优路径分析问题：

蚁巢有若干个储藏间，储藏间之间有路径相连。该图为 无向图，每条路径有相应的花费，路径正反方向通行的花费相同。要求从起点 N0 到终点 N17 的最优路径，并需要满足条件：
1.必须经过图中的绿色节点 N7、N12；
2.必须经过图中的两段绿色路段 (N2, N4)、(N13, N14)；
3.必须避开图中的红色路段 (N11, N12)；
求花费最少的最优路径。
说明：本案例来自西安邮电大学（XUPT）第12届数学建模竞赛赛题，本文进行了改编。



NetworkX是用于创建、操作和研究复杂网络动态、结构和功能的Python语言包。提供分析图的算法。图是由顶点、边和可选的属性构成的数 据结构，顶点表示数据，边是由两个顶点唯一确定的，表示两个顶点之间的关系。顶点和边也可以拥有更多的属性，以存储更多的信息。Networkx 支持创建简单无向图、有向图和多重图（multigraph）；内置许多标准的图论算法，节点可为任意数据，如图像文件；支持任意的边值维度，功 能丰富，简单易用。由于Networkx代码经过多次测试，性能方面也做了很多的工作，使用Networkx内置的多种图论算法能给开发SDN应 用带来很多的便利，可以节省开发时间，降低代码故障率。Networkx的经典图论算法可以解决最短路径问题。但是对于带有条件限制的最优路径问 题，则不能简单地通过Network X处理实现，需要引用函数，并进行适当的处理。

2. 设计方案

本程序的主要步骤分为：初始化；创建窗口，键入必经 点，必经边，禁止边；路径处理；求解条件最优路径；创建窗口选择条件；显示图像并输出。
旨在实现键入条件后，选择条件，画出最优路径。

2.1 初始化（创建无向图）

首先，建立一个无向图，并设立节点，给边赋代价值和 奖励值。运用NetworkX里的函数给边描边，进而实现一个带有代价值，奖励值的无向图。

2.2 创建窗口，键入必经点，必经边，禁止边

运用tkinter创建一个窗口，初始化窗口的文本和输入框。以此来获取入口键入值。

2.3 路径处理

获取文本框的键入值，并进行处理，改成与处理路径相配适的格式。处理无向图中的路径，使得其枚举出满足条件的路径。

2.4 求解条件最优路径

判断路径中必经边的两个顶点的路径是否相邻，即只选择含有必经边的路径。
计算路径的代价值和奖赏值，获取最小代价值及其路径，获取最大奖赏值及其路径。
并根据指定的代价值，在满足条件的路径中，获取最大奖赏值，即条件最优路径。

2.5 创建窗口选择条件

创建一个新的窗口，设置Button来选择条件，并运行相应的函数来显示图像。若选择关闭窗口，则结束程序。

2.6 显示图像并输出

对于得到的路径，用draw_networkx_edges画出路径，并保存相应的图片在子文件夹里，输出路径以及相对应的值。

3. 创新性描述

对于蚁穴问题，本代码在NetworkX原有的寻求最短路径的基础上实现以下创新功能：

3.1不同于简单路径规划问题，对于边加入了边权值，设置了代价值和奖励值，在此模型基础上，可以设置更多的属性，从而解决更加复杂的问 题。

3.2可以自行输入必经点，必经边，禁止边，并用tkinter创建窗口，更加灵活变通。

4.3路径处理

对于键入的值，首先要将字符串转化成数字，并对必经点必经边的顶点进行合并，转化成相应的格式。
处理路径：
4.3.1 首先，直接删除禁止边，并用all_simple_paths找到所有起点到终点的简单路径。
4.3.2 然后处理图像的必经点，选择满足路径中含有必经点和必经边的顶点的路径进行循环选择。
4.3.3 同时，关闭原来的窗口。
代码如下：
